There is always one more thing to add, before a thing is 'ready'. But that means nothing is ever ready.

Perry.el, my org-mode static site builder is far from ready, but this version more or less works, so here is version 0.0.1

Static websites and org-mode

There are several other implementations of a static site generator for org-mode, these are the ones that influenced me most:

  • Writing a static site generator using org-mode.
  • kelvinh/org-page: A static site generator based on Emacs and org mode.
  • emacs-china/EGO: EGO is a static site generator that depends on Emacs, Git and Org-mode.

Jekyll and org-mode

Why did I decide to build this one? First, coding something is a great way to learn, and this is my way of learning elisp.

Second, I think the best static site builder is Jekyll. It is quite simple, versatile, and immensely popular. I want to combine the good parts of Jekyll, with org-mode's integration in Emacs.

The plan is to make it easy to separate html and css in templates (using mustache), and make it easy to choose different templates for different pages on your site. Deployment will be automated, for starters to Github.
